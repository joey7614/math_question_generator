# Math Question Generator — Project Rules

## Project Goal
Generate SAT / AMC / Math Olympiad style questions from a sample input question.
Secondary: allow random generation by category (e.g. "give me 10 SAT algebra questions").

---

## Architecture Rules (DO NOT DEVIATE)

### Layer 1 — Pure Logic (NO AI allowed)
- Question text parsing (regex, keyword detection)
- Geometry keyword detection (triangle, circle, angle, etc.)
- LaTeX formatting and rendering
- Matplotlib figure rendering from JSON spec
- Hint retrieval (graph lookup by depth)
- Difficulty classification (derived from topic depth in graph)
- All graph traversal operations

### Layer 2 — AI (Gemini API only, 2 call types)

#### Call Type A: Topic Identification (cheap — use Flash model)
- Input: raw question text + predefined topic list
- Output: MUST be structured JSON only: `{"topics": ["topic_id_1", "topic_id_2"]}`
- AI picks from a predefined list — never free-form topic naming
- Keep prompt short and constrained

#### Call Type B: Question Generation (main call — use better model if needed)
- Input: topic list + difficulty + category + optional geometry flag
- Output: structured JSON: `{"question": "...", "answer": "...", "latex": "...", "figure_spec": {...} | null}`
- `figure_spec` is a structured geometry description (see Geometry Rules below)

### Layer 3 — Pure Logic (post-AI)
- Parse and validate AI JSON output
- Graph traversal for prerequisite breakdown
- Rendering pipeline (geometry + LaTeX)

---

## Knowledge Graph Rules
- Topic prerequisites are HARDCODED in `topic_graph.py` — never fetched from AI
- Each topic node contains: `id`, `description`, `prerequisites: []`, `categories: []`
- Traversal is recursive with max depth limit (default 4)
- Categories: `"SAT"`, `"AMC"`, `"olympiad"` — only these three
- Difficulty levels: `"basic"`, `"intermediate"`, `"advanced"` — derived from graph depth, not AI

---

## Geometry Rules
- AI outputs a `figure_spec` JSON — never raw matplotlib code (security risk)
- `figure_spec` schema:
  ```json
  {
    "type": "triangle | circle | quadrilateral | coordinate_plane",
    "vertices": {"A": [x, y], "B": [x, y], ...},
    "angle_labels": [{"vertex": "A", "label": "90°"}, ...],
    "side_labels": [{"from": "A", "to": "B", "label": "5"}],
    "right_angle_at": "A | null",
    "extra_labels": [{"position": [x, y], "label": "..."}]
  }
  ```
- `geometry_renderer.py` converts this spec to a matplotlib figure — pure logic
- All angle/side labels support LaTeX strings (e.g. `"x^2 + 2x"`)

---

## AI Provider
- Provider: Google Gemini (not Claude/Anthropic)
- Package: `google-generativeai`
- Topic identification: `gemini-2.0-flash` (cheap)
- Question generation: `gemini-2.0-flash` (can upgrade to `gemini-2.0-pro` if quality insufficient)
- API key: read from environment variable `GEMINI_API_KEY` — never hardcoded

---

## Output Format
- Questions are always rendered with LaTeX math (inline: `$...$`, block: `$$...$$`)
- Geometry figures saved as PNG to `./output/figures/`
- Final question output is a `Question` dataclass (see `models.py`)

## Coding Rules
- All AI responses MUST be validated against expected JSON schema before use
- Never execute AI-generated code (no `eval`, no `exec`)
- All dataclasses must have type annotations
- Functions longer than 40 lines should be split
- No hardcoded strings for topic IDs — always reference `topic_graph.TOPICS` dict keys
- Math expressions in questions must be wrapped in LaTeX delimiters before output
- Always handle AI JSON parse failures gracefully with a clear error message
- Use `Literal` types for constrained string fields (category, difficulty, figure type)

---

## What NOT to Do
- Do NOT use AI for graph traversal or hint generation
- Do NOT let AI output free-form topic names (always constrained to known topic IDs)
- Do NOT execute matplotlib code generated by AI
- Do NOT use Claude/Anthropic API (use Gemini)
- Do NOT store API keys in code
- Do NOT skip JSON schema validation on AI responses
- Do NOT invent new topic IDs at runtime — only use ones defined in `topic_graph.py`
